# 1. 数据类型

# 1.1 值类型

​ 看值类型之前，了解到 c# 中的八种整数类型，分别是 sbyte，byte，short，ushort，int，uint，long，ulong ，基类库为 System.Int 。

# 1.2 浮点类型

​ 浮点数的精度由有效数位的个数决定的，从 .Net Core 3.0 开始， double （双精度）型浮点可以表示的有效数数子最多为 17 位， float 型为最多九位（有一个前提，是该浮点数不是由字符串转换得来的），浮点数类型采用二进制浮点数来存储数值，这种方法会在小数部分产生舍入的误差问题，所以在实际的运算之中，用十进制和二进制所表示的实数都存在舍入的误差问题，作为对比， c# 的 decimal 类型在其所允许的进度范围之内不会有误差的

# 1.3 decimal 类型

​ c# 提供了 128 位精度的十进制浮点类型，它适合大而精确的计算，适用于金融的计算，decimal 类型保证范围内所有十进制数都是精确的，即 0.1 就是 0.1，而不是近似值，虽然精度高，但是范围小，是在一个可控制的范围，所以就会出现浮点数类型转换位 decimal 类型发生溢出的错误

# 1.4 布尔类型

​ c# 的第四个基元类型是 布尔类型 （Boolen），允许的值包括关键字 true 和 false 其基类为 System.Boolen ，理论上一个二进制位足以容纳一个布尔类型的值，但 bool 实际上的大小为一个字节

# 1.5 字符类型

​ 字符类型 char 表示 16 位字符，取值的范围对应于 Unicode 字符集，从技术上说，char 的大小和 16 位无符号整型（ushort）相同，但是 char 是 c# 的特有类型，要单独的对待。char 类型的基类为 System.Char ，有的字符不可以直接插入源代码，需要再进行特殊处理，首先输入反斜杠（\）前缀，再加上特殊的字符代码。反斜杠加上特殊字符代码统称为转义序列。

# 1.6 字符串

​ 零或者是多个字符的有限序列称为字符串，c# 的字符串是 string 类型，基类为 System.String ，string 是一种不可变类型。

#### 1.61 字面值

​ 字面值字符串输入代码，需要将文本放入到双引号中（"Hello，World"），c# 允许在字符串前使用 @符号，指明转义序列不被处理，结果是一个 逐字字符串字面值 ，它会将反斜杠当做普通的字符，还会逐字解释所有的空白字符

//在Unity中也是遵守这个条件的 
void Start()
{
    //在双引号之前加上@，就可以将字符串原样输出，转移字符将不处理
    string str = @"\n\t$%^&*(((((**&^&^^^ ";
    Debug.Log(str);
}

​ 假如同一个字符串字面值在程序集中多次出现，编译器在程序集中只定义字符串一次，且所有变量都指向它，如此，假如在代码中多次插入包含大量字符的同一个字符串字面值，最终的程序集只反映其中一个的大小。要想链接字符串就需要用 + 号操作符。

# 1.62 字符串插值

​ 字符串可以用插值技术嵌入表达式，语法就是在字符串前添加 $ 符号，并在字符中用一对大括号嵌入表达式，如下所示

void Start()
{
    string name = "rainbow";
    int age = 15;
    string sex = "女";
    //使用$来实现字符串的格式化
    string desc = $"我是{name},今年{age}岁了,我是{sex}孩子喔！";
    Debug.Log(desc);
}

​ 而且，逐字和插值可组合使用，但是要先指定 $，再指定 @，适用场景为字符串太长，需要字符逐字输出，那么就可以做到，稍微修改一下代码

void Start()
{
    string name = "rainbow";
    int age = 15;
    string sex = "女";
    //使用$来实现字符串的格式化,加上@逐字输出
    string desc = $@"我是{name},今年{age}岁了,
               ** **
             **  *  **
            ***     ***
             **     **
              **   **
                ***
            我是{sex}孩子喔！";
    Debug.Log(desc);
}

​ 字符串插值的内部工作原理就是调用了 string.Format() 方法的 语法糖 ，如下面代码所示，两个的效果是一模一样的

void Start()
{
    string name = "rainbow";
    int age = 15;
    string sex = "女";
    //$语法糖其实是调用了string.Format()
    object[] args = new object[] {name, age, sex};
    string desc = string.Format("我是{0},今年{1}岁了, 我是{2}孩	  子喔！", args);
    Debug.Log(desc);
}

# 1.63 字符串方法

​ string 类型也提供了几个方法来格式化，链接和比较字符串，具体的可以查看下面的 Api 以及示例，有了字符串插值的时候， string.Format() 就用的比较少了，但是在幕后，编译成 CIL 后都转换为调用 string.Concat() 和 string.Foramat() 来处理字符串的字面值。

​ 上表列出的都是静态方法，这意味着为了调用方法，需要在方法名之前附加方法所在的类型，（实例方法调用就不以类型名作为前缀了，而是以变量名字，或者对实例的其他引用）作为前缀。

高级主题：using 和 using static 指令

​ 之前调用静态方法需要附加命名空间和类型名前缀， using static 指令添加到文件顶部，每次使用程序集的方法时候，直接使用其中命名的方法即可， using static只支持静态方法或者属性，不支持实例方法 。

​ using 是直接忽略掉命名空间前缀，使用 using 指令时，不管是在实例化时，还是在调用静态方法时，还是在 c#6.0 新增的 nameof 操作符的时候，都可以省略对命名空间的引用，因为 using作用于整个文件或者是命名空间中 。

# 1.64 字符串格式化

​ 要在插值或者格式化的字符串中添加实际的左右大括号可以连写两个大括号表示

void Start()
{
    string name = "rainbow";
    int age = 15;
    string sex = "女";
    //使用$来实现字符串的格式化
    object[] args = new object[] {name, age, sex};
    string desc = string.Format("我是{{{0}}},今年{{{1}}}岁了, 	我是{{{2}}}孩子喔！", args);
    Debug.Log(desc);
}

# 1.65 换行符

​ 跨平台的话要依赖 System.WriteLine() 和 System.Environment.NewLine 而不是 \n 来确保跨平台兼容.

# 1.66 字符串长度

​ 判断字符串长度可以使用 string 的 Length 成员（或者叫 字段 ），该成员是 只读属性 ，不能设置，调用时也不需要任何的参数，如下面代码所示，字符串的长度不能直接设置，因为字符串的长度是更具字符数计算出来的，而且字符串是不可变的

void Start()
{
 	string str = "rainbow_ovo";
    Debug.Log($"The str`s length is {str.Length}");   
}

# 1.67 字符串长度不可变

​ string 类型由于性能的问题，一旦定义就无法更改，只能新开辟空间，而且旧的字符串也不可以更改，即使使用 text.ToUpper() 也只是返回一个新的字符串而已，用一个新的变量保存，原来的字符串还在，在内存中并没有改变。

​ 如果要大量修改字符串，就可以考虑使用 System.Text.StringBuilder 类型，而不是 string 类型，因为 StringBuilder 的实例方法会修改 StringBuilder本身的数据，不返回新的字符串 ，也就不会有多余的内存占用。
